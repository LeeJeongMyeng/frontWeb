[1단계:개념] 1. form의 속성을 예제와 함께 기술하세요.

속성


1) name: 폼의 이름을 지정

2) action

- 폼 데이터를 처리할 웹 서버 응용프로그램(jsp,php,asp등)
요청값을 서버단(백엔드)에서 받을 수 있다.
- submit 버튼을 클릭하면 브라우저는 action에 지정한 웹서버
응용프로그램을 실행한다.

3) method속성

- 폼데이터를 웹 서버로 전송하는 방식
- 전송방식 내용
default : get 방식으로 설정하지 않으면 이방식인데 url주소창에 요청값을
key=value 형식으로 보이게 한다.
post : 설정을 하면 url 주소창에는 보이지 않지만 내부적으로 요청값을 key=value형식으로 전송한다.

4) target

- frame단위나 새로운 윈도우창에 전송할 때, frame처리시 설정했던 옵션을 활용해서 처리한다.
_top,_parent,_blank

5) enctype

- 폼 데이터를 웹서버로 전송할때, 암호화 방식이 필요한 경우에 사용된다.
파일 전송시도 binary 데이터를 보내야겠기에 이 부분 속성을 설정한다.

1. input 태그

1) 대부분은 input 태그로 시작하여 type의 속성에 따라 2) type 속성 text : 한줄 텍스트 입력 창 ex) password : 암호 임력을 위한 한줄 텍스트 입력창 button : 단순 버튼 hidden : 데이터는 보이지 않지만 전송할 필요가 있을 때, 활용 ex) 기능버튼 등록/수정/삭제이 있지만, 요청값에 의해서 해당 내용을 전송해야할 때, 활용 된다. 즉, 요청값이 표시가 되지는 않지만 그 요청값이 프로세스에 따라 보이지 않게 전송될 때 활용된다. checkbox : 체크박스(다중선택) [] radio : 라디오 박스(단일선택) () file : 파일 선택 month|week|date|time|datetime-local : 년,월,일,시간등의 입력창 number|range : 스핀 버튼과 슬라이드바로 편리한 숫자 입력창 color : 색상 선택 창 email|url|tel|search : 이미지,url, 전화번호, 검색키워드 등 형식 검사 기능을 가진 텍스트 입력창 submit : form전체 데이터를 서버에 전송하는 버튼 reset : form전체 데이터를 입력 중, 초기화가 필요할 때, 클릭 시 초기화 처리
